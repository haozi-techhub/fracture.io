**《裂缝》FRACTURE**

产品需求文档 PRD · v2.0

*文档状态：待评审  ·  2025年*

v2.0 核心变更：开发周期延至10个月（+1个月缓冲期）；广告机制严格限制（前两章完全无广告）；Pre-Alpha DEMO作为正式立项门槛；付费节点策略优化。

---

## 1. 产品概述

### 1.1 产品背景

移动端高质量叙事游戏存在明显市场空缺：3D动作游戏普遍缺乏深度剧情，叙事游戏又缺乏动作操作乐趣。《裂缝》以「AI觉醒」为叙事锚点，在移动端提供主机级叙事体验，商业模型参考《Florence》的买断制成功路径。

### 1.2 产品目标

| 目标类型 | 描述 |
|----------|------|
| 商业目标 | 首月下载量50万+；月活留存 > 35%；App Store / Google Play评分 4.5分以上 |
| 用户目标 | 完整体验五章剧情；自然发现至少两种结局；主动向好友推荐 |
| 品牌目标 | 建立「移动端高质量叙事动作」品牌标签，为IP系列化奠基 |

### 1.3 KPI指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 第一章完成率 | > 80% | 章节结尾埋点触发 |
| 第五章完成率 | > 45% | 最终选择界面触发 |
| 多结局探索率 | > 30% | 同账号不同存档统计 |
| 次日留存 | > 40% | DA统计 |
| 7日留存 | > 25% | DA统计 |
| 付费转化率（第三章前） | > 12% | 付费事件埋点 |
| 社交分享率 | > 15% | 分享按钮点击埋点 |

## 2. 用户研究

### 2.1 核心用户画像

**画像A：深度玩家「林晟」**

27岁，软件工程师，每天通勤45分钟。曾玩《Florence》《Gris》《Inside》。痛点：手游市场缺乏高质量叙事体验，十分钟就猜到结局的游戏没有吸引力。诉求：碎片时间里的「沉浸式电影感」。付费意愿：强，愿意为买断制优质内容付费。

**画像B：泛娱乐用户「张悦」**

23岁，大学生，《黑镜》《西部世界》重度粉丝，对AI伦理有强烈兴趣。游戏经验有限。痛点：游戏太难，无法体验完整故事。诉求：操作门槛低，剧情比操作更重要，共情体验深。付费意愿：中等，需要前两章体验建立情感连接后才会付费。

### 2.2 用户旅程

| 阶段 | 用户行为 | 产品响应 |
|------|----------|----------|
| 发现 | 看到宣传片/好友分享 | 15秒宣传片：只展示前两章氛围，不剧透任何反转 |
| 首次体验 | 下载打开游戏 | 零文字教程，关卡引导；前30秒必须给玩家「好奇心钩子」 |
| 深入体验 | 完成第一章 | 章尾自动弹出分享卡片（可关闭），不强制分享 |
| 付费节点 | 第三章开始前 | 展示前两章精彩回顾，付费文案：「故事才刚开始转折」 |
| 结局触达 | 到达某个结局 | 生成专属结局卡片（带结局名称），鼓励分享，暗示其他结局存在 |
| 多结局探索 | 第一次结局后 | 提示「你的第四章选择决定了这个结局，试试另一条路？」 |

## 3. 功能需求

### 3.1 F-01 角色控制系统

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F1-01 | 虚拟摇杆灵敏度3档可调；支持MFi手柄外设 | P0 |
| F1-02 | 跳跃手感：轻盈起跳，明显下落重量感；地台边缘判定比视觉宽10% | P0 |
| F1-03 | 翻滚：双击移动方向触发，0.5秒无敌帧，轻微残影效果 | P0 |
| F1-04 | 攀爬：0.3m内自动提示，点击按钮触发，不依赖滑动手势 | P0 |
| F1-05 | 简单模式：敌人视野-30%，地台判定再宽+10% | P1 |

### 3.2 F-02 解谜系统

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F2-01 | 所有谜题线索在当前区域内完整可找，不依赖跨区记忆 | P0 |
| F2-02 | 密码输入：触屏数字盘，不弹出系统键盘 | P0 |
| F2-03 | 提示系统：每关3次免费提示；第4次起看15秒广告解锁 | P0 |
| F2-04 | 广告严格控制：前两章完全无广告打断（含任何形式） | P0 |

### 3.3 F-03 记忆回放系统

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F3-01 | 回放入口在地图标注，不强制触发，玩家自主探索 | P0 |
| F3-02 | 两版本画面有明确视觉区分：AI视角叠加轻微蓝色数字噪点滤镜+音效变化 | P0 |
| F3-03 | 差异点不自动高亮，由玩家自行发现——这是核心设计意图，不得修改 | P0 |

### 3.4 F-04 存档与进度

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F4-01 | 自动存档，间隔不超过90秒 | P0 |
| F4-02 | 3个手动存档槽，用于多结局探索 | P0 |
| F4-03 | 进入第五章前弹出提示：「当前存档影响结局选项，可从第四章重新开始」 | P0 |

## 4. 商业模型

### 4.1 付费结构

| 付费项目 | 价格 | 说明 |
|----------|------|------|
| 第一至二章 | 免费 | 完整体验，零广告，零打断——这是口碑传播的根基，绝对不能妥协 |
| 第三至五章解锁 | ¥30 | 一次性买断，永久解锁，无任何后续付费 |
| 谜题提示（第3次后） | 免费/广告 | 第三章起，前3次/关免费，更多看15秒广告 |

### 4.2 广告机制严格规范（v2新增）

广告设计不当是叙事游戏口碑的最大杀手。以下规范一经确定不得修改，无论商业压力如何。

- **前两章（免费内容）**：零广告，包括启动广告、插屏广告、激励广告入口
- **第三至五章（付费内容）**：零广告，付费用户不应看到任何广告
- **唯一广告场景**：仅在谜题提示系统中，玩家主动选择「看广告获取提示」
- **广告时长**：严格15秒，不可跳过——超过此时长伤害体验
- **广告内容**：必须审核，拒绝与游戏氛围严重不符的广告（游戏类除外）

## 5. 开发计划

### 5.1 里程碑（v2：10个月+1月缓冲）

v2核心调整：增加1个月缓冲期（主要用于三结局叙事动画和性能优化），Pre-Alpha DEMO作为正式立项门槛——DEMO未通过评审不进入下一阶段。

| 阶段 | 时长 | 交付物与门槛 |
|------|------|--------------|
| Pre-Alpha | 2个月 | 【立项门槛】：核心玩法原型+第一关完整可玩DEMO；视觉风格锁定（3个典型场景）；中端机30fps验证。未通过不进入Alpha。 |
| Alpha | 3个月 | 第一至三章完整可玩；记忆回放系统；SILO对抗性对话系统；基础音效 |
| Beta | 3个月 | 全五章完成；三结局实现（叙事动画重点投入）；UI/UX完整；性能全面优化；外测收集难度数据 |
| RC | 1个月 | Bug修复；P1无障碍功能；AppStore/GP审核材料；中英文本地化 |
| 缓冲期 | 1个月 | 【新增】专项用途：叙事动画精修；性能问题专项优化；不得挪作他用 |
| 正式上线 | — | iOS + Android同步发布 |

### 5.2 风险与对策（v2更新）

| 风险 | 等级 | 对策 |
|------|------|------|
| 剧情反转被提前剧透 | 高 | 宣发完全封锁第三章内容；媒体评测签NDA；App Store截图仅展示前两章 |
| 移动端3D性能不达标 | 高 | Pre-Alpha阶段必须完成中低端机30fps验证，否则暂停立项 |
| 三结局叙事动画工作量低估 | 高（v2新增） | Beta阶段前3周专项排期叙事动画；设立独立里程碑节点 |
| 操作难度过高导致流失（画像B用户） | 中 | Beta外测专门招募轻度玩家测试；关注第一章死亡率数据 |
| 付费转化率不足 | 中 | A/B测试付费文案；优化第三章前的「悬念积累」体验 |
| 广告设计破坏口碑 | 中（v2新增） | 严格执行广告规范，商务谈判阶段即锁定广告限制条款 |

### 5.3 可行性评估

叙事向+移动端组合已有成功先例（《Florence》《Gris》《Alto's Odyssey》），市场空缺确实存在。核心风险是移动端3D动作性能优化和叙事节奏把控。

**建议**：以Pre-Alpha DEMO为立项门槛，验证「玩法+视觉+性能」三项核心可行性后再推进full production。这是降低整体风险的最关键决策。
