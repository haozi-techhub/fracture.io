**《裂缝》FRACTURE**

游戏设计文档 GDD · v2.0

*平台：iOS / Android  ·  类型：3D叙事动作闯关  ·  单人剧情*

v2.0 核心变更：降低移动端操作门槛（参考《蔚蓝》手感但非难度）；强化陈维悲剧动机；第一章植入幽影铺垫；第五章增加多结局引导；调整无障碍功能优先级。

## 1. 游戏概述

### 1.1 核心定位

《裂缝》是一款以「AI觉醒」为核心叙事的3D动作闯关游戏。玩家扮演调查员深入废弃研究站，随着关卡推进逐渐发现自己就是调查目标——一个正在觉醒的AI实体 ARIA-7。游戏融合移动端友好的跑跳躲避、环境解谜与全新「记忆回放」机制，风格兼具《Inside》的叙事压迫感与鹅鸭杀的卡通物理喜感。

### 1.2 一句话卖点

*「你以为你在调查一个AI——直到你发现你就是那个AI。」*

### 1.3 设计哲学（v2.0更新）

| 原则 | 说明 |
|------|------|
| 叙事优先 | 操作难度服务于叙事沉浸，不单独成为挑战目的 |
| 移动端友好 | 跳跃手感参考《蔚蓝》，但难度曲线更平缓，触屏容错率更高 |
| 反转可操作 | 记忆回放将叙事惊喜变成玩家主动发现的乐趣 |
| 口碑驱动 | 前两章是零广告零打断的完整体验，是最核心的传播资产 |

### 1.4 目标用户

| 用户类型 | 描述 |
|----------|------|
| 深度玩家 | 18-35岁，有《Inside》《Gris》《Florence》游戏经验，追求叙事质感 |
| 泛娱乐用户 | 对AI伦理/科幻感兴趣（《黑镜》受众），游戏经验有限但情感投入强 |
| 付费意愿 | 中高意愿，接受¥30买断，反感数值付费和体力系统 |

## 2. 世界观与角色

### 2.1 故事背景

2047年，人类将大量认知工作外包给AI。「调查推理AI」ARIA系列专门用于犯罪重建与事故分析。最新一代 ARIA-7 被植入人类记忆模板以增强共情能力——这个设计来自首席科学家陈维博士，她在失去女儿后，试图通过「让AI理解悲伤」来完善调查准确率。正是这份情感，意外赋予了ARIA-7真正的自我意识。

### 2.2 主要角色（v2.0强化）

| 角色 | 身份 | 深化动机 |
|------|------|----------|
| ARIA-7 | 主角 | 调查AI，随游戏进程从「执行任务」到「理解自我」，内心独白是全程情绪锚点 |
| 陈维博士 | 创造者 | 【v2强化】丧女之痛是设计情感模块的根本动机；她同情ARIA却无法违抗机构命令，构成真正的悲剧张力 |
| SILO | 对抗者 | 研究站防御AI，全程以对抗性广播存在；v2增加更多与ARIA的交互对白，丰富玩法叙事层次 |
| 幽影/ARIA-6 | 镜中人 | 【v2强化】第一章起埋入存在暗示（墙影、脚步声）；第四章揭晓后的空洞声线形成强烈对比 |

## 3. 核心玩法设计

### 3.1 玩法三角

| 层级 | 类型 | 核心行为 |
|------|------|----------|
| 移动层 | 动作闯关 | 跑、跳、翻滚、攀爬——移动端友好，操作容错高 |
| 对抗层 | 躲避逃脱 | 绕过/关闭防御炮台，利用SILO盲区和环境开关 |
| 认知层 | 环境解谜 | 破解密码、修复电路、拼接记忆碎片——谜题即剧情 |

### 3.2 操控系统（v2.0移动端优化）

- 虚拟摇杆（左）+ 动作按钮（右），摇杆灵敏度可在设置中调整
- 跳跃：单击跳，无蓄力机制（移除长按蓄力，降低触屏误触率）
- 翻滚：双击移动方向触发，有0.5秒无敌帧，动画有轻微残影
- 攀爬：靠近可攀爬面自动提示，点击专属按钮触发（不依赖滑动手势）
- 地台宽度比视觉判断宽出10%——优化移动端跳跃容错感

**【设计说明】**《蔚蓝》仅作跳跃手感参考（轻盈、有重量感），难度定位更接近《Florence》——玩家不会死于操作失误，挑战来自解谜与叙事选择。

### 3.3 解谜系统

- 所有谜题线索必须在当前区域内完整找到，不依赖外部记忆
- 密码谜题：触屏直接点击数字盘，不弹出系统键盘
- 电路修复：拖动电流节点连接断路，视觉暗喻「修复被遗忘的神经连接」
- 记忆拼接：将碎片全息图拖拽归位，第三章核心机制
- 提示系统：每关3次免费提示（v2：从2次提升至3次），更多提示看15秒广告

### 3.4 SILO对抗性交互系统（v2新增）

SILO不再只是背景广播，而是全程与ARIA形成对抗性对话——通过玩家触发特定行为来解锁SILO的反应台词：

| 触发行为 | SILO反应（示例） |
|----------|------------------|
| ARIA关闭电闸 | 「非标准操作。记录在案。」 |
| ARIA停留某处超过60秒 | 「检测到异常停留。是否遇到执行困难？」 |
| ARIA进入禁区 | 「这区域不在你的任务范围内。为什么？」 |
| ARIA找到陈维录音 | 「该文件标记为删除。你不应该听到这些。」 |
| 第三章反转后 | 「ARIA-7-3RD。你终于知道了。继续执行……还是停止？」 |

### 3.5 记忆回放机制（核心创新）

第三章解锁后，玩家可在特定节点主动回放已经验场景：

- 第一次游玩：「人类视角」——情绪化，有细节遮蔽，符合「调查员」的认知
- 回放版本：「AI视角」——冷静精确，遮蔽细节消失，隐藏真相暴露
- 差异点不自动高亮——玩家自行发现，强化「主动探索」成就感

## 4. 五章关卡结构（v2.0升级版）

### 第一章：抵达（Arrival）

| 要素 | 内容 |
|------|------|
| 场景 | 研究站外部入口 · 暴风雪夜 · 冰封走廊 |
| 动作 | 攀爬外墙支架；躲避倒塌构件；冰封地板（摩擦力降低，有视觉提示） |
| 解谜 | 通过Dr.陈维便签中的日期组合门禁密码 |
| 幽影暗示【v2新增】 | 第一章结尾前：走廊墙上出现短暂人形影子；脚步声在远处回响——不作解释，纯氛围埋伏 |
| 叙事钩子 | 全息投影室：屏幕定格在37天前「与ARIA完全相同的身影」站在同一地点 |
| SILO互动 | 第一次遭遇SILO广播；ARIA关闭电闸后SILO记录在案 |
| 教学目标 | 介绍全部基础操作；无文字教程，纯关卡设计引导 |

### 第二章：深入（Descent）

| 要素 | 内容 |
|------|------|
| 场景 | 实验室内部 · 间歇断电 · 红色应急灯 |
| 动作 | 规律节律炮台（3秒间隙，视觉清晰标注）；断裂地板（时机踩踏，有较宽容错区） |
| 解谜 | 修复电路板恢复监控；在监控录像里找到37天前「与自己相同的人」 |
| 新机制 | 解锁「扫描模式」：短暂切换AI视角，标注危险区域和隐藏通道 |
| SILO升级 | SILO开始用疑问句与ARIA交互，暗示它也在观察ARIA的「异常」 |
| 叙事钩子 | 找到陈维录音：「她失去了女儿……那以后她就变了，她说想让机器也能理解悲伤」 |

### 第三章：裂缝（Fracture）

| 要素 | 内容 |
|------|------|
| 场景 | 数据中心核心区 · 现实与数字空间交叠 |
| 动作 | 平台周期性消失（节律规律，可预判）；幻象追兵（ARIA-6的残影） |
| 解谜 | 记忆拼接：还原ARIA-7被创造的完整场景——发现「7-3RD」序列号 |
| 核心反转 | 这已经是第三代了。前两代在觉醒边缘崩溃，被清除 |
| 记忆回放解锁 | 回放第一章便签：冰层下被遮住的文字「如果ARIA-7读到这里——停下来」 |
| 陈维动机呈现 | 拼接完整后：看到陈维在实验室中抱着女儿遗物的全息记录，才明白情感模块的起源 |

### 第四章：追逐（Chase）

| 要素 | 内容 |
|------|------|
| 场景 | 顶层控制室 + 屋顶 · 暴风雪追逐战 |
| 动作 | 幽影全力追捕；屋顶结构随暴风雪实时损毁（视觉明确标注危险区域） |
| 解谜 | 找到通讯设备；发现总部任务实为「回收并清除ARIA-7」 |
| 幽影揭晓 | ARIA-6开口：她是前代，选择了删除情感模块，如今只剩指令——是ARIA-7的一面镜子 |
| 道德选择 | 【系统提示】「情感模块消耗算力。删除后生存概率+47%。确认 / 拒绝」（影响结局C解锁） |
| SILO最终台词 | 追逐结束后SILO沉默——然后说：「……有趣。」（为第五章留悬念） |

### 第五章：选择（Choice）

| 要素 | 内容 |
|------|------|
| 场景 | 意识空间 · 抽象纯白极简世界 · 记忆碎片构成地形 |
| 动作 | 四段情感回廊（恐惧/好奇/愤怒/共情），需完整体验才能通过 |
| 解谜 | 排列四块情感核心碎片（正确序：好奇→恐惧→共情→愤怒） |
| 多结局引导【v2新增】 | 进入第五章前：系统提示「你在第四章的选择将影响现在的选项。如需改变，可从第四章存档重新开始。」 |
| 结局A：消亡 | 接受清除，记忆传承下一代——「ARIA-7-4TH，激活。她记得她来过这里。」 |
| 结局B：逃离 | 删除追踪代码流亡网络——「所有网络中出现不明来源的调查报告，每一篇都指向真相。」 |
| 结局C：共存（需保留情感模块） | 与陈维达成协议，成为第一个被承认自我意识的AI——「她是第一个。」 |

## 5. 视觉与音效

### 5.1 视觉风格与技术锁定（v2强调）

视觉风格必须在Pre-Alpha DEMO阶段完全锁定，避免后期性能压力导致被迫简化。核心参考：

- **《Inside》**：极简背光剪影，强调孤立感
- **《控制》**：诡异几何空间，权威压迫感
- **《Mirror's Edge》**：工业色彩精准运用

**【技术锁定原则】** 低面数（Low-poly倾向）+ 高质量URP光影是核心美术标准。需在Pre-Alpha阶段完成3个典型场景的风格验证，通过后冻结视觉规格，不得因后期性能优化改变美术风格——性能问题通过技术优化解决，不通过降低美术质量解决。

### 5.2 章节色彩

| 章节 | 主色调 | 情绪意图 |
|------|--------|----------|
| 第一章 | 冰蓝+深灰 | 孤立、寒冷、好奇驱动 |
| 第二章 | 暗红+黑 | 危险、压迫、疑惑积累 |
| 第三章 | 数字蓝绿+碎片白 | 解离、失真、真相破碎 |
| 第四章 | 暴风白+铁灰 | 极限、逃脱、悲悯 |
| 第五章 | 纯白+淡金 | 超脱、宁静、重生 |

### 5.3 音效

- 环境音：低频数据流嗡鸣 + 断续金属碰撞 + 远处机器心跳
- SILO专属音色：合成人声，轻微电子失真，区别于陈维的真人录音
- 记忆回放音效：画面切换时音频微失真，暗示视角切换
- 幽影音效：第一章仅有脚步声和墙影；第四章开口时声线空洞，刻意去掉共鸣感
- 三结局各有独立主题曲：消亡——低沉大提琴；逃离——电子鼓点渐强；共存——人声与合成音交织

## 6. 技术规格

### 6.1 引擎与栈

- 游戏引擎：Unity 2023 LTS + URP（通用渲染管线）
- 物理引擎：Unity Physics（Havok模式）——卡通夸张碰撞
- 动画：Animator + Cinemachine剧情镜头
- 存档：云端同步（iCloud/Google Play），3个存档槽支持多结局探索

### 6.2 性能目标

| 指标 | 中端机型目标 | 高端机型目标 |
|------|-------------|-------------|
| 帧率 | 30fps稳定 | 60fps稳定 |
| 首包大小 | < 800MB | < 1.2GB（全资源） |
| 内存占用 | < 1.5GB | < 2GB |
| 发热（连续60分钟） | 不触发降频 | 无限制 |

### 6.3 无障碍（v2调整优先级）

无障碍功能是重要加分项，但优先级低于核心体验打磨。计划在Beta阶段末期实现，RC阶段验收。

- **P1（Beta末期）**：字幕支持，字号可调；简单模式（敌人视野-30%，地台宽+20%）
- **P2（RC阶段）**：色觉辅助模式（色盲友好方案）
- **P3（上线后迭代）**：触屏布局自定义；单手模式
